# Shopify テーマ開発ガイド（Horizon）

## テーマ情報

- **テーマ:** Horizon v1.0.2 (Shopify)
- **構造詳細:** [`theme-profile.md`](theme-profile.md) を必ず参照すること
- **デザイン指針:** [`DESIGN.md`](DESIGN.md) — 新しいデザイン・UIを作るときは必ずこのデザイン言語（カラー/タイポグラフィ/余白/コンポーネント）に従う

## 実装前に必ずすること

1. `theme-profile.md` でCSS設計・命名規則・Liquidパターンを確認する
2. デザイン作業のときは `DESIGN.md` のトークン（色・余白・角丸・ボタン等）に沿わせる
3. 変更対象のファイルを読む
4. 既存パターンに合わせて実装する（独自スタイルを持ち込まない）

---

## 開発ワークフロー（4段階）

メインエージェントのコンテキストを汚さないため、複雑なタスクはサブエージェントで分担する。

| フェーズ | エージェント | 用途 |
|---------|-------------|------|
| 調査 | `research` | コードベース理解・バグ原因特定 |
| 計画 | `plan` | 実装手順・変更ファイルの設計 |
| 実装 | `implement` | コード変更・ファイル作成 |
| 検証 | `test` | 整合性チェック・品質確認 |

---

## スキル一覧

| コマンド | 用途 |
|---------|------|
| `/analyze-project` | テーマ全体を分析して theme-profile.md を更新 |
| `/bug-hunt` | バグの原因調査と修正方針の提示 |
| `/perf` | パフォーマンスボトルネックの特定と改善案 |
| `/refactor` | コード品質改善（動作は変えない） |
