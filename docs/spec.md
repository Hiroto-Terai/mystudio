# 仕様書: hirot デモストア ポートフォリオ仕上げ

作成日: 2026-07-16 / 対象: mystudio テーマ（Dawn 15.5.0 + hirot カスタム層）

## 1. プロダクト概要

hirot はフリーランスのポートフォリオとして人に見せるためのスキンケア EC デモストア。
本仕様は、テーマコード（Liquid / JSON / CSS）だけで完結する残タスクを対象に、「見せられる完成度」に到達させる。
Admin API・外部サービス・新規 JS は使わない。

## 4. 受け入れ基準

すべて `shopify theme dev --store xn-0nsp96djkas7be59f.myshopify.com` を起動し、http://127.0.0.1:9292 をブラウザで開いて検証する。

### F1: 商品カードの不正な HTML 構造の修正

1. トップページの商品グリッドで、任意の商品カードの画像・商品名・価格のいずれをクリックしても、その商品の詳細ページへ遷移する。
2. カード内の「カートに入れる」表示は修正前と同じ位置・見た目で表示されている（ホバー時の挙動も従来どおり）。
3. ブラウザの開発者ツールで商品カードの DOM を確認したとき、`a` 要素の内側に `button` 要素（またはリンク等の別のインタラクティブ要素）が存在しない。
4. About ページ（/pages/about）の商品グリッドでも 1〜3 が同様に成立する。

### F2: About ページの実文言化

1. /pages/about をブラウザで開き、ページ内検索（Cmd+F）で「プレースホルダ」を検索してヒットが 0 件である。
2. 差し替え後の文言はいずれも日本語として自然で、スキンケアブランド「hirot」の設定（国産植物由来・シンプル処方・敏感肌向け）と矛盾しない。
3. 「無添加◯項目」のような未確定表現（◯・×・TBD 等）が画面上に残っていない。数値を出す箇所は具体的な数値になっている。
4. ページ全体をスクロールして、文言差し替えによるレイアウト崩れ（見出しの折り返し破綻、カードの高さ不揃いの悪化など）が起きていない。

### F3: コレクション一覧ページのブランド演出（ヘッダー）

1. /collections/all をブラウザで開くと、ページ上部にトップページと同系統の hirot ブランド演出（配色・タイポグラフィ・見出しの様式）が表示され、素の Dawn のバナーとの見た目の違いが一目で分かる。
2. コレクション名（および説明文があれば説明文）がページ上部に表示される。
3. 既存のフィルター・並び替え・ページネーションが従来どおり操作できる（フィルターで絞り込みができ、並び替えで順序が変わる）。
4. ブラウザ幅を 960px 前後・640px 前後まで縮めても、ヘッダー部分が崩れず読める状態を保つ。

### F4: コレクション一覧の商品グリッドのトーン統一

1. /collections/all の商品カードが、トップページの商品カードと同系統の見た目（配色・角丸・余白・ホバー時の雰囲気）に見える。
2. 商品カードをクリックすると商品詳細ページへ遷移する。
3. フィルターで絞り込んだ後・並び替えを変えた後も、カードの見た目が 1 の状態を保っている。
4. ブラウザ幅を 640px 前後まで縮めたとき、カードが破綻せずに閲覧できる。

## 5. スコープ外

- Admin API・外部サービス連携（レビューアプリ、メール等）を使う機能
- 商品データ・metafield・metaobject の登録/変更（ストア側データ作業）
- カート・チェックアウト機能の拡張（「カートに入れる」を実際にカート追加する JS 実装を含む。hirot 層は JS ゼロ方針）
- 多言語対応（schema・文言は日本語ハードコードの既存方針を維持）
- Unsplash フォールバック画像の実画像への差し替え（データ作業のため）
- Dawn 標準セクション・スニペット・JS 本体の改変（リスキンは既存方針どおり `hirot.css` の上書きレイヤーで行う範囲に留める）
- About 以外の固定ページ（contact 等）の新規作成・文言整備
