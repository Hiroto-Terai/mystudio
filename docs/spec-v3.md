# 仕様書 v3: アプリレス機能群 & Flow 自動化ショーケース

作成日: 2026-07-17 / 対象: mystudio テーマ（Dawn 15.5.0 + hirot カスタム層）/ 前提: spec v1・v2（docs/spec.md, docs/spec-v2.md）完了済み

## 1. 概要

「Shopify アプリで月額課金されがちな定番機能を、テーマネイティブで自作できる」ことを営業材料として見せるための機能群を追加する。あわせて、コードでは実現しない運用自動化を Shopify Flow で構成できることを紹介するショーケースページを設ける。

既存の hirot デザイン言語（`--hirot-*` トークン・Hanken Grotesk / Noto Sans JP のタイポグラフィ・角丸・halo シャドウ・eyebrow/heading の見出しパターン）と UI/UX を絶対に崩さないこと。新規 UI はすべて既存セクションと並べて違和感がない見た目に揃える。

### 前提（本仕様全体に適用）

- **JS ポリシーの拡張（二層ルール）**: 従来の「hirot 層は JS ゼロ」を本機能群から次の二層に変更する。
  - **表示系セクション**（FAQ、Flow ショーケース、レビュー一覧などの静的表示）: 引き続き JS ゼロ。
  - **インタラクティブ機能**（お気に入り、プログレスバー、配送日時指定、最近見た商品、フォーム制御など）: Dawn 流の自己完結 Web Component を許可。外部ライブラリ依存ゼロ・対象セクションからのみ読み込むこと。
- **ゲスト体験前提**: localStorage を使う機能（お気に入り・最近見た商品）はログイン不要で動作するよう設計する。顧客アカウントとの連携・同期はスコープ外。
- **データ/管理画面作業の分担**: metaobject 定義（`review` / `faq`）とエントリ投入、ページ（wishlist / automation）の作成、Shopify Flow の実自動化セットアップは、テーマ実装ではなくオーケストレーター（または人間）がストア管理画面側で行う。本仕様の「必要なデータ定義」に従うこと。
- **デザイン分担**: 詳細なデザイン仕様・モックは designer エージェントが別途作成する。本仕様は「何が見える / 何ができる」に集中する。
- **既存との整合**: 送料無料しきい値は **¥5,000**（trust-strip の「¥5,000以上で送料無料」訴求と一致させる）。metaobject の落とし穴（エントリがアクティブでないとテーマから見えない）は `staff_member` と同様に `review` / `faq` にも適用される。

## 2. 機能一覧

| # | 名前 | 1行説明 | 優先度 |
|---|------|---------|--------|
| F1 | お気に入り（wishlist） | 商品カードと商品詳細にハートを置き、localStorage に保存。ヘッダーに件数付きアイコン、専用ページで一覧・削除できる | P0 |
| F2 | 送料無料プログレスバー | カート（ドロワー/ページ）に「あと¥X,XXXで送料無料」のプログレスバーを表示。しきい値 ¥5,000 | P0 |
| F3 | 配送日時指定 | カートページで配送日（翌日〜14日先、除外曜日は設定可）と時間帯を選択し、注文に引き継がれる cart attributes として記録する | P1 |
| F4 | レビュー表示・投稿 | 商品詳細に metaobject `review` ベースのレビュー一覧（星・名前・本文・日付）と、Flow 承認フローに乗る体裁の投稿フォームを置く。既存の aoi.rating 表示と整合させる | P1 |
| F5 | 最近見た商品 | 商品詳細の下部に、localStorage ベースの「最近見た商品」の商品行を表示する | P2 |
| F6 | 再入荷通知 | 売り切れ商品の商品詳細に「再入荷をメールでお知らせ」フォーム UI を表示する（Flow 連携の入口として位置づけるデモ） | P2 |
| F7 | ギフトラッピング | カートに「ギフトラッピングを希望する（+¥300）」チェックを置き、cart attribute として記録する（実課金はデモでは行わない） | P2 |
| F8 | FAQ アコーディオン | metaobject `faq` 駆動のアコーディオンセクション。JS なしの開閉 UI で任意ページに配置できる | P2 |
| F9 | Flow 自動化ショーケース | 「Flow でこんな自動化が作れる」紹介ページ。5 つの自動化例をカード + インライン SVG 図解（hirot トークン配色）で見せる | P3 |

## 3. スプリント計画

| スプリント | 名前 | 含む機能 | 内容 | 依存 |
|-----------|------|---------|------|------|
| Sprint 1 | お気に入り | F1 | 二層ルール初の Web Component を含む最大の機能。ハート・ヘッダーカウント・一覧ページまで一式 | なし |
| Sprint 2 | カート訴求バンドル | F2, F7 | カートの小粒 2 機能（プログレスバー + ギフトラッピング）をまとめて実装 | なし |
| Sprint 3 | 配送日時指定 | F3 | カートページの配送日・時間帯選択と attributes 記録。Sprint 2 で整えたカート UI の並びに追加する | Sprint 2 |
| Sprint 4 | レビュー | F4 | metaobject `review` の一覧表示・平均整合・投稿フォーム。事前にデータ定義とサンプルエントリの投入が必要 | データ定義（後述） |
| Sprint 5 | PDP 回遊強化 | F5, F6 | 商品詳細ページの 2 機能（最近見た商品 + 再入荷通知）。最近見た商品は F1 と同じ localStorage パターンを踏襲する | Sprint 1 |
| Sprint 6 | コンテンツショーケース | F8, F9 | JS ゼロの表示系 2 本（FAQ アコーディオン + Flow 自動化ページ）。F9 は F1/F6 の機能を自動化例として言及するため最後に置く | Sprint 1, 5 |

### 必要なデータ定義（オーケストレーター/人間の事前作業）

- **metaobject `review`**: `product`（商品参照）/ `rating`（整数 1〜5）/ `author`（単一行テキスト）/ `body`（複数行テキスト）/ `date`（日付）。ストアフロント公開を有効にし、デモ用に複数商品へ 2〜3 件ずつアクティブなエントリを投入する。
- **metaobject `faq`**: `question`（単一行テキスト）/ `answer`（複数行テキスト）。ストアフロント公開を有効にし、アクティブなエントリを 5 件以上投入する。
- **ページ**: 「お気に入り」ページ（ハンドル `wishlist`）と「自動化ショーケース」ページ（ハンドル `automation`）を作成し、対応するテンプレートを割り当てる。
- **Shopify Flow**: 実ワークフローの構築はテーマ外の作業（本仕様のスコープ外）。F9 はあくまで紹介コンテンツであり、Flow 未構築でも完結する。
- **商品データ**: F6 の検証用に、在庫 0（売り切れ）の商品を最低 1 つ用意する。

## 4. 受け入れ基準

すべて `shopify theme dev --store xn-0nsp96djkas7be59f.myshopify.com` を起動し、http://127.0.0.1:9292 をブラウザで開いて検証する。

### F1: お気に入り（wishlist）

1. TOP やコレクションページの商品カード、および商品詳細ページにハートの切り替え UI が表示され、クリックでオン/オフでき、ページをリロードしても状態が保持されている。
2. お気に入りに商品を追加すると、ヘッダーのお気に入りアイコンに件数バッジが即座に反映される（0 件のときはバッジが目立たない/非表示）。
3. /pages/wishlist を開くと、お気に入り登録した商品が既存の商品カードと同じ見た目で一覧表示され、各カードから削除すると一覧とヘッダー件数の両方が更新される。
4. シークレットウィンドウ（未ログイン）でも 1〜3 がすべて成立する。

### F2: 送料無料プログレスバー

1. 商品をカートに入れてカートドロワーを開くと、小計 ¥5,000 未満のとき「あと¥X,XXXで送料無料」の文言とバーが表示され、X,XXX が実際の差額と一致している。
2. 商品を追加して小計が ¥5,000 以上になると、バーが満タンになり「送料無料」達成を示す文言に切り替わる。
3. カートページ（/cart）でも 1〜2 と同じ表示・挙動が確認できる。
4. カート内の数量を変えると、ページ全体をリロードしなくても（またはリロード後に）バーと差額表示が新しい小計に追随する。

### F3: 配送日時指定

1. /cart に配送日の選択 UI があり、選べる日付が「翌日から 14 日先まで」の範囲に限られている（当日や 15 日以降は選べない）。
2. テーマエディタで除外曜日を設定すると、その曜日が選択肢から消える（または選択できない）。
3. 時間帯として「指定なし / 午前中 / 14-16時 / 16-18時 / 18-20時」相当の選択肢から選べる。
4. 配送日と時間帯を選んだ状態で http://127.0.0.1:9292/cart.js を開くと、attributes に選択した配送日・時間帯が記録されている。

### F4: レビュー表示・投稿

1. レビューエントリが登録された商品の詳細ページに、レビュー一覧（星・投稿者名・本文・日付）が投入済みエントリどおりに表示される。
2. 同じページの商品名付近の星評価・件数表示が、レビュー一覧から計算される平均・件数と矛盾しない（一覧のある商品では一覧基準の値が表示される）。
3. レビュー投稿フォームに星・名前・本文を入力して送信すると、「承認後に掲載されます」の趣旨の完了状態になる（送信内容が即座に一覧へ公開されないことがデモの正しい挙動）。
4. レビューが 1 件もない商品の詳細ページでは、壊れた表示（空の星・0 件の羅列など）にならず、レビューがない旨または投稿導線のみが自然に表示される。

### F5: 最近見た商品

1. 商品詳細ページを 3 商品以上順に閲覧した後、別の商品詳細ページの下部に「最近見た商品」の行が表示され、直前に見た商品が新しい順に並ぶ。
2. 現在表示中の商品自身は「最近見た商品」の行に含まれない。
3. 行内のカードをクリックするとその商品の詳細ページへ遷移し、シークレットウィンドウ（閲覧履歴なし）ではこの行自体が表示されない。

### F6: 再入荷通知

1. 売り切れ（在庫 0）の商品詳細ページに「再入荷をメールでお知らせ」のフォーム（メールアドレス入力 + 送信）が表示される。
2. 在庫のある商品の詳細ページにはこのフォームが表示されない。
3. メールアドレスを入力して送信すると、受付完了の状態（「再入荷時にお知らせします」の趣旨）が同じページ上で確認できる。

### F7: ギフトラッピング

1. カート（ドロワーとページの少なくとも両方で導線が成立する箇所）に「ギフトラッピングを希望する（+¥300）」のチェック UI が表示される。
2. チェックを入れた状態で http://127.0.0.1:9292/cart.js を開くと、attributes にギフトラッピング希望が記録されており、チェックを外すと記録が解除される。
3. チェック状態はカートページをリロードしても保持されている。

### F8: FAQ アコーディオン

1. FAQ セクションを配置したページで、投入済みの `faq` エントリが質問の一覧として表示され、質問をクリックすると回答が開閉する。
2. ブラウザの JavaScript を無効にした状態でも開閉が機能する。
3. 見出し・余白・配色が既存 hirot セクション（eyebrow + 見出しのパターン）と揃って見える。

### F9: Flow 自動化ショーケース

1. /pages/automation を開くと、「お気に入り追加→通知」「再入荷→通知配信」「購入後レビュー依頼メール」「VIP 顧客自動タグ付け」「高リスク注文の自動保留」の 5 例がカードで表示される。
2. 各カードにトリガー→アクションの流れがわかる図解が含まれ、図解の配色が hirot のブランド配色（緑系トークン）で統一されて見える。
3. ページ内に外部画像への依存がなく（図はページに埋め込まれた SVG）、ブラウザ幅を 960px 前後・640px 前後に縮めてもカードと図が崩れない。
4. お気に入り・再入荷通知のカードから、ストア内の該当機能（wishlist ページ・売り切れ商品の通知フォーム等）へ体験導線（リンク）がある。

## 5. スコープ外

- 顧客アカウントとお気に入り/閲覧履歴の連携・デバイス間同期（localStorage のゲスト体験のみ）
- ギフトラッピングの実課金（ラッピング用商品・手数料の請求はデモでは行わない。attribute 記録まで）
- 再入荷通知・レビュー投稿の実バックエンド処理（メール実配信、レビューの自動掲載・承認システムの実装。Flow 側の実構築はテーマ外作業）
- Shopify Flow ワークフローの実セットアップ（F9 は紹介コンテンツであり、Flow 未構築でも完結する）
- レビューの並べ替え・ページネーション・画像付きレビュー
- 有料アプリ・外部サービス・外部ライブラリの導入
- 多言語対応（schema・文言は日本語ハードコードの既存方針を維持）
- チェックアウト画面以降のカスタマイズ（配送日時・ラッピングの attributes が注文へ引き継がれる標準挙動を利用するのみ）
- Dawn 標準セクション・スニペットの改変を伴う最適化（既存の分離方針を維持）
